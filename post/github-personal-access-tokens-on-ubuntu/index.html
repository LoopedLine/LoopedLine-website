Creating and Saving GitHub Personal Access Tokens on Ubuntu Linux [SOLVED]

2021 August 18
software development Git GitHub Ubuntu Linux technical troubleshooting technical troubleshooting - solved

The Problem

Pushing repository changes to GitHub via the command line no longer works without a Github Personal Access Token.

The Process

On 2021-08-13 GitHub no longer allowed authentication by account password for Git operations on GitHub.com.

The error notifications were clear, as were the steps for creating a personal access token.

The recommendations for Caching your GitHub credentials in Git on Linux required further investigation to confirm their suitability and to confirm the specific steps required for installing on specific flavors of Linux.

The StackOverflow discussion of Where to store the personal access token from GitHub? was helpful, especially this answer.

Installing Git Credential Manager Core appeared to be the best option.

The Solution

Per the instructions for installing on Other Ubuntu/Debian distributions:

Download the latest .deb package, and run the following:

sudo dpkg -i <path-to-package>

Then:

git-credential-manager-core configure

From there, according to the documentation:

Once it’s installed and configured, Git Credential Manager Core is called implicitly by Git. You don’t have to do anything special, and GCM Core isn’t intended to be called directly by the user.*
